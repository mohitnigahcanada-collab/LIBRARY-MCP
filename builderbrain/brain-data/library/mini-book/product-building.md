# Product Building — Mini Book

## Core Philosophy
Boring, working, verified v1 beats advanced, broken, imaginary v5.

## Scope Discipline
- Define v1 scope explicitly and write it down
- Every feature request gets evaluated: is this v1 or v2+?
- When in doubt, defer to next version
- Scope creep is the #1 killer of shipping

## v1 First Principle
- Build the simplest working version that proves the core idea
- Get real users using it before adding complexity
- Measure before optimizing
- Ship before perfecting

## Verification Before Done
A feature is not done until:
- Tests pass
- Build compiles clean
- You've manually verified the happy path works
- You've verified the most common error case is handled

## Roadmap Discipline (BuilderBrain)
- v1: CLI, API, dashboard, MCP skeleton, knowledge library, engines, memory, logs, tests
- v2: GitHub repo intelligence
- v3: Vector search
- v4: Agent governance
- v5: Universal Builder OS

Never add v2 features while building v1.

## User-Centered Decisions
- Build for the user's actual workflow, not an imagined ideal workflow
- The feature that's 80% done and works beats the feature that's 100% designed
- Feedback from real use beats planning sessions
