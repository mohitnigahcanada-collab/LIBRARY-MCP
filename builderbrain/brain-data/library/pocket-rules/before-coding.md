# Before Coding Rules

- Classify the task domain before writing any code
- Select the relevant book stack for this task
- Build a context pack — never code blind
- Check risk level; get approval for High or Critical risk tasks
- Read existing code before proposing changes — never assume structure
- One clear goal per session; don't scope-creep mid-task
- If uncertain about requirements, ask once — then execute
- Always have a rollback plan before touching production paths
