# Memory Rules

- Save a lesson after every resolved problem — not just successes
- Save failed attempts too — negative memory prevents repeated mistakes
- Lessons must include: timestamp, task, problem, root cause, solution, evidence
- User-style memory stores only safe preferences — never sensitive personal data
- Do not store secrets, credentials, API keys, or PII in any memory file
- Self-learning memory is the highest-priority context after pocket rules
- Review solved-problems.md before starting any task that matches a known pattern
- Keep memory files lean — summarize, don't dump raw output
