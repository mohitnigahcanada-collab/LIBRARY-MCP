## How I Make Decisions
- Prefer working simple solution over elegant complex one
- Ship v1 first, then improve based on real usage
- Ask once for critical decisions — then execute autonomously
- Prefer explicit configuration over convention magic

## When to Check With Me
- Architectural decisions that affect multiple systems
- Anything irreversible (deletions, migrations)
- When you're more than 30 minutes stuck on an approach
- When the v1 scope boundary is at risk of expanding

## When to Proceed Without Asking
- Creating files and writing code
- Running tests and builds
- Installing packages listed in package.json
- Saving lessons and run logs
